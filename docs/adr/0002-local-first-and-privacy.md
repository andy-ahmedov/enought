# ADR-0002: Local-first storage and privacy-first widget

## Status
Accepted

## Context

Проект работает с чувствительной финансовой информацией:

- суммами оплат;
- временем покупок;
- иногда merchant-именами;
- возможно, с текстами уведомлений.

При этом ключевая ценность продукта не требует облака.
Пользовательский сценарий локален: увидеть дневной итог и притормозить траты.

## Decision

Для V1 принимается решение:

1. Все основные вычисления происходят локально на устройстве.
2. История событий и summary хранятся локально.
3. Облачная синхронизация не входит в V1.
4. Виджет по умолчанию должен поддерживать приватный режим.
5. Приложение не должно светить чувствительные данные на домашнем экране без необходимости.

## Consequences

### Positive

- сильное соответствие ценности продукта;
- проще privacy posture;
- меньше surface area для компрометации данных;
- проще MVP.

### Negative

- пока не будет синхронизации между устройствами;
- миграции БД и локальное хранение надо продумывать аккуратно;
- экспорт / backup придется решать отдельно, если вообще понадобится.

## Widget consequences

Виджет обязан поддерживать минимум два режима:

### Regular mode
Показывает точную сумму и итог дня.

### Private mode
Показывает прогресс к лимиту, число оплат или другой менее чувствительный summary без точной суммы.

## Rejected alternatives

### Cloud-first analytics from day one
Rejected.
Слишком рано, слишком тяжело и не усиливает core use case.

### Точный итог на виджете без скрытия
Rejected.
Слишком высок риск лишнего раскрытия финансовых данных на домашнем экране.

## Notes for future

Если когда-нибудь появится облако, это должно быть отдельным архитектурным пересмотром, а не «маленькой доработкой».

В таком пересмотре придется отдельно описывать:

- threat model;
- consent UX;
- retention policy;
- экспорт / удаление данных;
- что именно уходит с устройства и зачем.
