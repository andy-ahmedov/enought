# ADR-0003: Explicit V1 exclusions

## Status
Accepted

## Context

Если не зафиксировать явные исключения, проект быстро расползется в одну из сторон:

- обход платежного стека;
- общий учет всех расходов;
- ранний облачный продукт;
- опасные permission-based shortcuts.

Поэтому нужно явно перечислить, что **не входит** в V1.

## Decision

В V1 не входят:

1. Прямой доступ к внутренностям Mir Pay.
2. Reverse engineering Mir Pay.
3. NFC/HCE interception как источник данных.
4. Собственный кошелек / wallet role.
5. Accessibility-based scraping чужих экранов.
6. SMS как основной источник событий.
7. Root / Xposed / модифицированные системные сценарии.
8. Серверная синхронизация и пользовательские аккаунты.
9. Полноценный общий трекер всех расходов.
10. Токсичные behavioral mechanics, которые стыдят пользователя.

## Consequences

### Positive

- сохраняется фокус продукта;
- резко упрощается MVP;
- снижается риск платформенных и policy-проблем;
- архитектура остается объяснимой.

### Negative

- часть возможностей сознательно откладывается;
- точность распознавания придется добирать через parser + manual correction;
- проект может казаться менее «магическим», чем хотелось бы.

## Guardrail

Если новая идея нарушает этот ADR, действовать так:

1. не внедрять ее напрямую;
2. оформить предложение и причины;
3. описать выгоды и риски;
4. дождаться решения пользователя.
